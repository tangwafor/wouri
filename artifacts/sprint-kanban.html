Wouri

Six sprints, start to finish. Every one ends on four green gates.

Roughly 28 weeks across six phases. Nothing moves to the next sprint until the build is green, the end-to-end sweep is green, the machine self-test is green, and a Fabrice UAT run is green. The investor and customer pitches grow one slice per sprint, so the story is always current and always demonstrable.

01The definition of done, every sprint

These four gates are not a phase at the end. They are the exit condition of every sprint, and three of the four are living artifacts that grow with the product rather than one-off tasks. The Fabrice UAT reuses the Human Test Suite already standing (dev-backed staging, seeded role bench, realtime triage feed), so "a test for Fabrice as we go" needs no new infrastructure.

1. Build green

Typecheck, migrations apply clean, no regulatory literal in code, no policy-less table, no unbound document field, no em-dash.

check-canonicals.mjs + tsc

2. End-to-end sweep

The whole app walked as each role at phone size, screenshotted, asserting real behaviour. Fails on any console error or any 5xx. Tenant B must see none of tenant A.

ui-qa-sweep.mjs + jest-expo

3. Machine self-test

Core flows in rollback subtransactions plus RLS probes run as anon and as a wrong-tenant JWT. Grows one flow set per sprint. A red here blocks the merge.

wouri-selftest (in-DB)

4. Fabrice UAT

A human tester walks the new sprint on dev-backed staging, following a per-sprint catalog, submitting pass or fail into a realtime feed the build agent watches and fixes.

tester portal + qa_live_triage

A fifth, softer gate rides along: the pitch touchpoint. Each sprint firms one part of the investor deck and one line of the customer one-pager, so a demo is possible from Sprint 3 and the raise story is complete by Sprint 5.

03Post-launch backlog: the network turns on

The tenant network mechanisms enter in leverage order after the core ships, each on one of three isolation-safe paths (platform-owned common good, opt-in consent, or anonymized aggregate). The registry common good is already live from Sprint 0; the verification mark from Sprint 2. The rest follow.

Anonymized benchmarking

Quality, price differential, cost and discrepancy rates as percentile bands, emitted only at a cohort of 50 or more with noise. Never row-level.

Aggregated financing

Opt-in permissioned reads let a lender rely on the verified record. Wouri never lends and never matches.

Shared plot commons

A plot mapped once, reused with holder consent, governed by a cross-tenant yield-cap ledger that blocks EUDR declaration-in-excess laundering.

Counterparty registry

Reciprocal factual attestations with right of reply, matched by private set intersection so no one exposes their book.

Group consolidation

Diaspora holdings and cooperatives-of-cooperatives on organization_groups, one shared group certificate.

Second origin, second commodity

Timber deepened to processed wood; a second country added as pure registry inserts, proving the base was levelled.

04The pitches, and where they live

The investor pitch and the customer pitch are companion documents, not slides buried in a sprint. Each grows on the pitch touchpoint above: the customer headline exists from Sprint 0, the demo hooks land as the features do (offline verify in Sprint 2, the cockpit in Sprint 3, field capture in Sprint 4), and the investor deck is complete and demonstrable by the first-exporter milestone in Sprint 3, with the dual-rail proof by Sprint 5. Both are being drafted now with real market numbers and will arrive alongside this board.
